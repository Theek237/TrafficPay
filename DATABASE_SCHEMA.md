# MongoDB Database Schema

This document outlines the database schema. 

## ⚙️ Global Standards
* **Database Engine:** MongoDB 7.x with Mongoose 8.x.
* **Naming Convention:** `snake_case` collection names, `camelCase` field names.
* **Auth Tokens:** Single JWT access token with 24-hour expiry.
* **Soft Deletes:** Handled via `isActive: Boolean` flag.
* **Timestamps:** Auto-managed by Mongoose (`{ timestamps: true }`).

---

## 4. `fines` Collection
Core entity for issued traffic fines. GPS fields removed; location handled via an optional string.

| Field | Type | Req | Key | Notes |
| :--- | :--- | :--- | :--- | :--- |
| `_id` | ObjectId | | PK | Auto-generated by MongoDB. |
| `referenceNo` | String | Yes | UNQ | Pattern: TF-YYYYMMDD-NNNNNN. |
| `vehicleNo` | String | Yes | IDX | Uppercase (e.g., CAB-1234). |
| `officerId` | ObjectId | | FK | Refers to `users._id`. |
| `categoryId` | ObjectId | Yes | FK | Refers to `fine_categories._id`. |
| `districtId` | ObjectId | Yes | FK | Refers to `districts._id`. |
| `amount` | Decimal128| Yes | | Copied from category at issue time. |
| `status` | String | | IDX | Enum: `PENDING`, `PAID`, `OVERDUE`, `CANCELLED`. |
| `location.address`| String | | | Optional human-readable string. |
| `issuedAt` | Date | Yes | | Timestamp of issuance. |
| `dueDate` | Date | Yes | | `issuedAt` + 30 days. |

* **Status Flow:** `PENDING` -> `PAID` (on success) or `OVERDUE` (dueDate < now). `OVERDUE` -> `PAID` (late payment). `PAID` and `CANCELLED` are terminal states.
